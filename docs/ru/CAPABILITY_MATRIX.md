# Матрица возможностей

Эта матрица документирует, что UtahMosphere OS **v25.0** реализует сегодня, в отличие от маркетинговых материалов или планов будущих релизов. Используйте её для точных ожиданий при миграции и разработке.

---

## HTTP API — конечные точки

| Конечная точка | Метод | Статус | Примечания |
|----------------|-------|--------|------------|
| `/health` | GET | **Реализовано** | Проверка живости узла |
| `/status` | GET | **Реализовано** | Состояние UI, список арендаторов, статус claim |
| `/command` | POST | **Реализовано** | Выполнение голосового интента (тело JSON) |
| `/app/{name}` | GET | **Реализовано** | Доступ к приложению через Tycoon (402 до оплаты) |
| `/s3/*` | * | В планах | Описано в руководстве по миграции; маршруты ещё не подключены |
| `/lambda/*/invoke` | POST | В планах | Заглушки обработчиков создаются только при развёртывании |
| `/rds/read/*`, `/rds/write` | * | В планах | Реестр существует; HTTP-маршруты не подключены |

## Основные подсистемы

| Компонент | Статус | Что работает сегодня |
|-----------|--------|----------------------|
| **Ядро (`utahmosphere_os.py`)** | Реализовано | Реестр, голосовые интенты, манифесты маршрутов UtahX, gossip сети |
| **Quantum Ledger** | Реализовано | Корневой vibe claim, проверка биометрического хеша, открытый режим до claim |
| **Voice Bridge** | Реализовано | Google STT + извлечение MFCC vibe-print → `/command` |
| **Utah-Tycoon** | Частично | Генерация счетов, симулированное закрытие за 60 с, шлюз HTTP 402 |
| **UtahNetes Gossip** | Частично | UDP-мультикаст синхронизации арендаторов в LAN |
| **Global Swarm** | Частично | UDP-таблица пиров, ping keep-alive; полный Kademlia lookup — заглушка |
| **Демон Lazarus** | Частично | Добавляет комментарии патча в `handler.py` (не полная перезапись AST) |
| **UI Utah-Flux** | Реализовано | Панель Tkinter, читает `flux_ui_manifest.json` |
| **Прокси UtahX** | Частично | JSON-манифесты маршрутов записываются; живой TCP-прокси-процесс отсутствует |

## Варианты развёртывания

| Метод | Статус | Платформа |
|-------|--------|-----------|
| `python3 utahmosphere_os.py` | Реализовано | Все (локально задайте `UTAH_DATA_DIR`) |
| `python3 genesis_deploy.py` | Реализовано | Linux предпочтителен; Windows для разработки — ОК |
| `sudo bash setup.sh` | Реализовано | Linux (сервис systemd) |
| `docker-compose up` | Реализовано | Опционально; host networking |

## Модель безопасности

| Функция | Статус | Примечания |
|---------|--------|------------|
| Единый корневой владелец vibe | Реализовано | Первый говорящий при claim владеет узлом |
| Поле `authorized_nodes[]` | Заглушка | Хранится в JSON реестра; в коде не применяется |
| HMAC-подписи арендаторов | Документировано | Рецепт есть; принудительное применение в ядре — частично |
| Подпись Ed25519 | В планах | Упоминается в документации; не реализовано |
| `UTAH_SECRET_VECTOR` по умолчанию | Реализовано | Смените в продакшене |

## Связь с Docker / Nginx

**Основная среда выполнения** UtahMosphere — bare-metal Python. Docker и Nginx — **опциональные унаследованные пути**:

- `docker-compose.yaml` — удобная обёртка для локальных проб
- `nginx.conf` — эталонная конфигурация; суверенный путь — JSON-манифесты UtahX
- `setup.sh` — удаляет Docker/Nginx при чистой установке Linux (продакшен-суверенные узлы)

В гибридных средах можно держать Docker/Nginx рядом с UtahMosphere во время миграции.
