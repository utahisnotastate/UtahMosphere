# Движок запутанной дельты состояния (v32.0)

**Запутанная синхронизация состояния** передаёт только *математическую дельту* состояния реестра — позволяя глобальным узлам достичь идентичного состояния с **накладными расходами <1 КБ** вместо полных payload реестра.

## Модуль (`state_diff_engine.py`)

```python
delta = get_state_delta(local_state, remote_state)
merged = apply_state_delta(remote_state, delta)
```

| Функция | Назначение |
|---------|------------|
| `get_state_delta(local, remote)` | Минимальная дельта на уровне ключей |
| `apply_state_delta(base, delta)` | Восстановление синхронизированного состояния |
| `encode_delta(local, remote)` | Упаковка дельты и хешей для mesh |
| `state_hash(state)` | Канонический отпечаток SHA-256 |
| `should_use_delta(local, remote)` | Использовать дельту, если меньше полного JSON |

## Интеграция с mesh

UtahNetes gossip отправляет `registry_delta` вместо полного `registry`, когда это эффективнее по полосе. Узлы-свидетели проверяют `state_hash` перед слиянием.

## Переменные окружения

| Переменная | По умолчанию | Назначение |
|------------|--------------|------------|
| `UTAH_STATE_DIFF_ENFORCE` | `1` | Предпочитать дельта-синхронизацию |
| `UTAH_STATE_DIFF_MAX_BYTES` | `1024` | Максимальный размер payload дельты |
